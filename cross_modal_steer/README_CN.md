# 跨模态引导目录

本文件夹包含多模态混合专家模型的高级路由机制和跨模态对齐策略的探索性实现和分析。

## 📁 用途

此目录探索以下方面的替代架构：
- **路由机制**：将 token 路由到专家的不同方式
- **跨模态对齐**：桥接音频和文本模态的策略
- **架构权衡**：复杂度与表达能力的分析

⚠️ **注意**：这些是**研究探索**，可能不用于主训练流程。它们作为未来改进的参考实现。

## 📝 文件

### `router_comparison.py`

**用途**：实现并比较混合专家模型中专家路由的多种路由器架构。

**关键类**：

1. **CurrentRouter** - 简单基线路由器
   - 每个 token 单次路由决策
   - 使用 `nn.Linear` 计算每个专家的门控 logits
   - 通过 einsum 操作应用引导向量
   - 返回作为输入残差加法的引导输出

2. **HierarchicalRouter** - 两级分层路由
   - 全局路由器做出高级决策（默认 4 个全局专家）
   - 局部路由器（每个全局专家一个）做出细粒度决策
   - 结合全局和局部引导，带层归一化
   - 更具表达力但计算量更大

3. **ProgressiveRouter** - 多阶段渐进路由
   - 顺序通过多个阶段路由
   - 每个阶段从前一阶段细化路由决策
   - 阶段之间包含层归一化
   - 跟踪每个阶段的门控分数

4. **AttentionBasedRouter** - 基于注意力的路由
   - 使用多头注意力计算路由权重
   - 注意力头充当不同的路由视角
   - 组合注意力输出以进行最终路由决策
   - 更复杂的路由机制

**主要函数**：
- `compare_router_implementations()` - 创建虚拟输入并测试所有路由器，比较输出形状和参数计数
- `analyze_routing_patterns()` - 分析每种路由器类型的专家利用率和路由模式

**使用方法**：
```python
from cross_modal_steer.router_comparison import compare_router_implementations

# 比较所有路由器类型
compare_router_implementations()
```

### `cross_modal_steering.py`

**用途**：实现跨模态特征对齐和引导方法，用于桥接音频和文本模态。

**关键类**：

1. **CrossModalFeatureAligner** - 原则性特征空间对齐
   - 将音频和文本投影到共享隐藏空间
   - 执行交叉注意力（音频查询，文本键/值）
   - 投影回 LLM 兼容的文本维度
   - 使用残差连接和层归一化
   - 理论上最合理但计算量最大

2. **ModalitySpecificSteering** - 独立模态处理与融合
   - 音频和文本模态的单独路由器
   - 每个模态有自己的引导向量
   - 在共享融合空间中融合模态特定输出
   - 将融合输出投影到 LLM 空间
   - 高效且可解释

3. **CrossModalAdapter** - 轻量级适配器方法
   - 音频到文本空间的简单顺序适配器
   - 适配音频和文本之间的交叉注意力
   - 最小参数开销
   - 对轻量级集成有效

4. **ProgressiveCrossModalSteering** - 多阶段跨模态对齐
   - 通过多个阶段渐进细化
   - 每个阶段执行跨模态注意力
   - 逐步建立对齐
   - 最具表达力的方法

**主要函数**：
- `compare_cross_modal_approaches()` - 使用虚拟输入测试所有四种方法，比较输出形状、参数计数并提供见解

**使用方法**：
```python
from cross_modal_steer.cross_modal_steering import compare_cross_modal_approaches

# 比较所有跨模态方法
compare_cross_modal_approaches()
```

### `router_analysis_and_cross_modal_steering.md`

**用途**：路由器实现和跨模态引导策略的文档和分析。

**内容部分**：

1. **路由器实现比较**
   - 分析当前实现（简单、高效、表达能力有限）
   - 分层路由器（更适合复杂模式，成本更高）
   - 渐进路由器（多阶段细化，良好平衡）
   - 基于注意力的路由器（通过注意力进行复杂路由）

2. **跨模态对齐方法**
   - 特征空间对齐（原则性但昂贵）
   - 模态特定引导（高效且可解释）
   - 跨模态适配器（轻量级）
   - 渐进跨模态引导（最具表达力）

3. **建议**
   - 对于逐层引导：从渐进路由器开始
   - 对于跨模态对齐：使用特征空间对齐
   - 关键见解：冻结 LLM 解码器约束至关重要
   - 跨模态注意力需要适当的特征空间桥接

**关键见解**：
- 渐进路由器为复杂对齐模式提供更好的表达能力
- 特征空间对齐对多模态集成至关重要
- 没有对齐的直接交叉注意力是有问题的
- 计算成本与表达能力之间的权衡

## 🔬 研究方向

这些实现探索：

1. **路由器架构**：
   - 单阶段 vs. 多阶段路由
   - 扁平 vs. 分层专家组织
   - 基于注意力的路由机制

2. **跨模态对齐**：
   - 特征空间投影策略
   - 模态特定 vs. 共享路由
   - 渐进细化方法

3. **权衡**：
   - 参数效率 vs. 表达能力
   - 计算成本 vs. 对齐质量
   - 简单性 vs. 灵活性
