# 已弃用目录

本文件夹包含早期实验性实现和基线方法，这些已被主代码库中更高效或更有效的版本所取代。

## 📁 用途

这些文件记录了 SteerMoE 项目的演变过程，可作为：
- **历史参考**，用于理解设计决策
- **教育示例**，展示不同的架构方法
- **基线对比**，用于消融研究

⚠️ **注意**：这些实现**不用于当前实验**，但保留用于参考和可重现性。

## 📝 文件

### `aligner.py`

**用途**：带编码器后引导的基础 SteerMoE 实现。

**功能**：
- 实现 `SteerMoEAligner` - 在 Whisper 编码器之后应用引导向量的简单模块
- 使用单个路由器（线性层）为每个 token 计算门控分数
- 基于门控分数计算引导向量的加权和
- 将引导调整添加到原始音频表示

**弃用原因**：这是一个简化的仅编码器后方法。它缺乏逐层控制，被在每个编码器层应用引导以实现更细粒度控制的更复杂实现所取代。

---

### `layer_wise_aligner.py`

**用途**：带每层路由器和引导向量的逐层引导实现。

**功能**：
- 实现 `LayerWiseSteerMoEAligner`，在 32 个 Whisper 编码器层的每一层应用引导
- 为每层维护单独的路由器（总共 32 个路由器）
- 维护层特定的引导向量和缩放因子
- 通过编码器处理音频，在每层应用引导调整
- 包含比较分析，显示逐层引导与编码器后引导的优缺点

**弃用原因**：此方法计算成本更高（32 倍路由操作）且参数更多。文件本身包含的分析表明它可能过拟合且更难训练。被更高效的替代方案（如共享路由器方法）所取代。

---

### `layer_wise_whisper.py`

**用途**：直接集成逐层引导的修改版 Whisper 编码器包装器。

**功能**：
- 实现 `LayerWiseSteeringWhisperEncoder`，包装原始 Whisper 编码器
- 为每层添加引导向量和路由器
- 冻结原始编码器参数
- 在前向传播期间逐层应用引导调整
- 包含详细分析，比较逐层引导与编码器后引导方法
- 提供何时使用每种方法的建议

**弃用原因**：与 `layer_wise_aligner.py` 类似，这代表了逐层引导的早期尝试。文件自身的分析建议从编码器后引导开始，只有在需要特定好处时才转向逐层引导。被更高效的实现所取代。

---

### `shared_router_implementation.py`

**用途**：探索不同的路由器架构 - 比较多路由器 vs 单路由器 vs 带层特定权重的共享路由器。

**功能**：
- 实现 `SharedRouterLayerWiseSteeringWhisperEncoder` - 混合方法，使用：
  - 所有层共享一个路由器（减少参数）
  - 层特定权重矩阵来转换共享路由器输出
  - 层特定缩放因子
- 包含三种路由器方法的详细比较：
  1. **多路由器**（每层一个）- 最灵活但参数最多
  2. **单路由器** - 高效但灵活性较低
  3. **带权重的共享路由器** - 平衡方法
- 提供参数计数分析和计算复杂度比较

**弃用原因**：此文件似乎是探索性/分析性的，而非最终实现。它比较了各种方法并推荐单路由器方法以提高效率。共享路由器方法更复杂，但相比更简单的替代方案没有明显优势。

---

### `al_models.py`

**用途**：通过适配器结合 Whisper 编码器和 GPT-2 解码器的音频转文本模型。

**功能**：
- 实现 `AudioToText` - 端到端 ASR 模型
- 从 Whisper 编码器获取音频特征并投影以匹配 GPT-2 嵌入维度
- 将音频提示与文本 token 嵌入拼接
- 通过 GPT-2 传递组合嵌入以生成文本
- 包含贪婪解码的推理示例
- 处理训练（带标签）和推理模式

**弃用原因**：这是一个基础的音频转文本架构，不包含 SteerMoE 引导机制。它代表了在引导集成到流程之前的早期基线方法。被包含引导向量以实现更好音频-文本对齐的版本所取代。

---

## 🔍 关键见解

这些文件记录了 SteerMoE 项目的演变：

1. **早期阶段**：简单的编码器后引导（`aligner.py`）
2. **探索阶段**：逐层引导尝试（`layer_wise_aligner.py`、`layer_wise_whisper.py`）
3. **架构分析**：路由器设计比较（`shared_router_implementation.py`）
4. **基线模型**：无引导的基础音频转文本（`al_models.py`）

弃用表明项目转向了更高效和有效的实现，可能将这些实验的见解整合到更清晰的生产就绪版本中。

## 📊 与当前实现的比较

| 方面 | 已弃用版本 | 当前实现 |
|------|-----------|----------|
| **路由** | 多路由器（32×）或带权重的共享路由器 | 单一高效路由器 |
| **参数** | 1050 万（多路由器）或复杂混合 | 32.7 万（减少 32 倍） |
| **引导** | 编码器后或带单独路由器的逐层 | 带共享路由器的逐层 |
| **复杂度** | 更高，更多组件 | 更简单，更高效 |
| **性能** | 基线或实验性 | 优化且已验证 |
